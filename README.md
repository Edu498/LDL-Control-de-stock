Control de stock para local comercial
Presentado por: Ángela
Problema detectado
El local realiza ventas todos los días, pero no cuenta con un sistema que controle automáticamente el stock disponible. Esto dificulta saber cuándo un producto está por agotarse, cuándo debe realizarse un pedido al proveedor y qué productos necesitan reposición.
Objetivo del proyecto
Desarrollar un sistema de control de stock que permita registrar ventas, calcular existencias y generar alertas cuando los productos estén en bajo stock o sin disponibilidad.
Funciones principales
Registrar productos.
Registrar ventas.
Cargar inventario inicial.
Calcular stock disponible.
Registrar salidas de productos.
Mostrar alertas visuales cuando haya stock bajo.
Mostrar alertas cuando no haya stock disponible.
Generar pedidos al proveedor.
Revisar compras realizadas.
Realizar control físico periódico del inventario.
Roles sugeridos
Administrador del sistema.
Vendedor o usuario de carga diaria.
Encargado de compras o reposición.
Flujo básico sugerido
Se cargan los productos.
Se registra el stock inicial.
Se registran las ventas diarias.
El sistema descuenta automáticamente las unidades vendidas.
Cuando un producto llega al mínimo establecido, se muestra una alerta.
Se genera o sugiere un pedido al proveedor.
Se realiza control físico periódico para verificar diferencias.
Observaciones del docente
La idea es válida y tiene buen potencial como sistema de gestión inicial. Sin embargo, todavía falta desarrollar mejor las pantallas, el flujo completo del sistema y la estructura de trabajo.

Es importante definir qué datos tendrá cada producto, cómo se registrarán las ventas y cómo se calculará el stock mínimo.
Prioridad inicial
Comenzar por el diseño de las entidades principales:

Producto.
Categoría.
Venta.
Detalle de venta.
Proveedor.
Pedido.
Movimiento de stock.

Luego avanzar con las pantallas básicas de carga, consulta y alertas.
